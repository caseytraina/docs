## Trigger Workflow

Use this endpoint to initiate a run of one of your configured workflows.

### Endpoint

```http
POST https://api.joinoverlap.com/{orgId}/{projectId}/{workflowId}/trigger
````

* **Path Parameters**

  * `orgId` — Your organization’s ID.
  * `projectId` — The project under which the workflow resides.
  * `workflowId` — The specific workflow to trigger.

### Authentication

All requests must include a valid Bearer token:

```http
Authorization: Bearer YOUR_API_KEY
Content-Type: application/json
```

### Request Body

```json
{
  "url": "https://youtube.com/watch?v=dQw4w9WgXcQ",
  "options": {
    "priority": "high",
    "notify": true
  }
}
```

* `url` (string, required) — The resource URL the workflow should process.
* `options.priority` (string, optional) — One of `low`, `medium`, or `high`. Defaults to `medium`.
* `options.notify` (boolean, optional) — If `true`, send a notification when the run completes.

### Example Usage

```js
import requests

const orgId      = "org_2w4fxjp97l193RY9cwjvR6D3Ja8"
const projectId  = "0iYNwiRR7qU4VEw9SM1l"
const workflowId = "2339ea1e-35e4-49ca-9113-108ff2f14dfb"
const apiUrl     = `https://api.joinoverlap.com/${orgId}/${projectId}/${workflowId}/trigger`

const headers = {
  "Content-Type": "application/json",
  "Authorization": "Bearer YOUR_API_KEY"
}

const payload = {
  url: "https://youtube.com/watch?v=dQw4w9WgXcQ",
  options: {
    priority: "high",
    notify: true
  }
}

requests.post(apiUrl, { json: payload, headers })
  .then(res => console.log(res.data))
  .catch(err => console.error(err))
```

### Sample Response

```json
{
  "message": "Workflow trigger initiated successfully",
  "status": "pending",
  "triggerId": "trigger_20250624184340_3f3c921a"
}
```

* `message` (string) — Confirmation text.
* `status` (string) — Current status of the trigger (`pending`, `running`, `completed`, etc.).
* `triggerId` (string) — Unique identifier for this workflow run.
